Modern Semantic Elements

Semantic elements in HTML5 help bring structure and accessibility to web code. They are HTML tags that organize the HTML code and allow future developers (including oneself) to be able make sense of it when revising it.

Some more modern semantic elements you should use in your projects are:

  • Aside
  • Section
  • Header
  • Article
  • Time
  • Datalist
  • Audio

In the past, many people used basic <div> elements that simply say “there’s a structured element here”. They typically need to be followed with documentation explaining the object, and their use should be kept to a minimum. Using a modern tag like <section>, <article>, <aside>, or <header> immediately identifies the type of resource represented in a given space.

Some tags, like <datalist>, <audio>, or <canvas> also allow one to easily do useful things on a web page. Using non-deprecated elements not only saves developers the time and headache required to figure out what the important elements are and how to use them, but also further enables assistive, translational, and indexing technologies.
